# [Add Skills to Your Agent](https://ai-sdk.dev/cookbook/guides/agent-skills\#add-skills-to-your-agent)

In this guide, you will learn how to extend your agent with [Agent Skills](https://agentskills.io/), a lightweight, open format for adding specialized knowledge and workflows that load at runtime from markdown files.

At its core, a skill is a folder containing a `SKILL.md` file with metadata and instructions that tell an agent how to perform a specific task.

```typescript
1my-skill/

2├── SKILL.md          # Required: instructions + metadata

3├── scripts/          # Optional: executable code

4├── references/       # Optional: documentation

5└── assets/           # Optional: templates, resources
```

## [How Skills Work](https://ai-sdk.dev/cookbook/guides/agent-skills\#how-skills-work)

Skills use **progressive disclosure** to manage context efficiently:

1. **Discovery**: At startup, agents load only the name and description of each available skill (just enough to know when it might be relevant)
2. **Activation**: When a task matches a skill's description, the agent reads the full `SKILL.md` instructions into context
3. **Execution**: The agent follows the instructions, optionally loading referenced files or executing bundled code as needed

This approach keeps agents fast while giving them access to more context on demand.

## [The SKILL.md File](https://ai-sdk.dev/cookbook/guides/agent-skills\#the-skillmd-file)

Every skill starts with a `SKILL.md` file containing YAML frontmatter and Markdown instructions:

```yaml
1---

2name: pdf-processing

3description: Extract text and tables from PDF files, fill forms, merge documents.

4---

5

6# PDF Processing

7

8## When to use this skill

9Use this skill when the user needs to work with PDF files...

10

11## How to extract text

121. Use pdfplumber for text extraction...

13

14## How to fill forms

15...
```

The frontmatter requires:

- `name`: A short identifier
- `description`: Instructions for when to use this skill

The Markdown body contains the actual skill content with no restrictions on structure or content.

## [Prerequisites](https://ai-sdk.dev/cookbook/guides/agent-skills\#prerequisites)

To support skills, your agent needs:

1. **Filesystem access** to discover and load skill files (read files, read directories)
2. **A load skill tool** that reads the `SKILL.md` content into context
3. **Command execution** (optional) if skills bundle scripts (e.g. a full sandbox environment)

## [Step 1: Define a Sandbox Abstraction](https://ai-sdk.dev/cookbook/guides/agent-skills\#step-1-define-a-sandbox-abstraction)

This guide uses a generic sandbox abstraction for flexibility across
environments. If you're building for Node.js, you can use `fs/promises` and
`child_process` directly instead.

Create a generic sandbox interface that provides a consistent way to interact with the filesystem. This abstraction lets you implement it differently depending on your environment (Node.js fs, a containerized sandbox, cloud storage, etc.):

```ts
1interface Sandbox {

2  readFile(path: string, encoding: 'utf-8'): Promise<string>;

3  readdir(

4    path: string,

5    opts: { withFileTypes: true },

6  ): Promise<{ name: string; isDirectory(): boolean }[]>;

7  exec(command: string): Promise<{ stdout: string; stderr: string }>;

8}
```

## [Step 2: Discover Skills at Startup](https://ai-sdk.dev/cookbook/guides/agent-skills\#step-2-discover-skills-at-startup)

Scan skill directories and extract metadata from each `SKILL.md`:

```ts
1interface SkillMetadata {

2  name: string;

3  description: string;

4  path: string;

5}

6

7async function discoverSkills(

8  sandbox: Sandbox,

9  directories: string[],

10): Promise<SkillMetadata[]> {

11  const skills: SkillMetadata[] = [];

12  const seenNames = new Set<string>();

13

14  for (const dir of directories) {

15    let entries;

16    try {

17      entries = await sandbox.readdir(dir, { withFileTypes: true });

18    } catch {

19      continue; // Skip directories that don't exist

20    }

21

22    for (const entry of entries) {

23      if (!entry.isDirectory()) continue;

24

25      const skillDir = `${dir}/${entry.name}`;

26      const skillFile = `${skillDir}/SKILL.md`;

27

28      try {

29        const content = await sandbox.readFile(skillFile, 'utf-8');

30        const frontmatter = parseFrontmatter(content);

31

32        // First skill with a given name wins (allows project overrides)

33        if (seenNames.has(frontmatter.name)) continue;

34        seenNames.add(frontmatter.name);

35

36        skills.push({

37          name: frontmatter.name,

38          description: frontmatter.description,

39          path: skillDir,

40        });

41      } catch {

42        continue; // Skip skills without valid SKILL.md

43      }

44    }

45  }

46  return skills;

47}

48

49function parseFrontmatter(content: string) {

50  const match = content.match(/^---\r?\n([\s\S]*?)\r?\n---/);

51  if (!match?.[1]) throw new Error('No frontmatter found');

52  // Parse YAML using your preferred library

53  return yaml.parse(match[1]);

54}
```

## [Step 3: Build the System Prompt](https://ai-sdk.dev/cookbook/guides/agent-skills\#step-3-build-the-system-prompt)

Include discovered skills in the system prompt so the agent knows what's available:

```ts
1function buildSkillsPrompt(skills: SkillMetadata[]): string {

2  const skillsList = skills

3    .map(s => `- ${s.name}: ${s.description}`)

4    .join('\n');

5

6  return `

7## Skills

8

9Use the \`loadSkill\` tool to load a skill when the user's request

10would benefit from specialized instructions.

11

12Available skills:

13${skillsList}

14`;

15}
```

The agent sees only names and descriptions. Full instructions stay out of the context window until loaded.

## [Step 4: Create the Load Skill Tool](https://ai-sdk.dev/cookbook/guides/agent-skills\#step-4-create-the-load-skill-tool)

The load skill tool reads the full `SKILL.md` and returns the body (without frontmatter):

```ts
1function stripFrontmatter(content: string): string {

2  const match = content.match(/^---\r?\n[\s\S]*?\r?\n---\r?\n?/);

3  return match ? content.slice(match[0].length).trim() : content.trim();

4}

5

6const loadSkillTool = tool({

7  description: 'Load a skill to get specialized instructions',

8  inputSchema: z.object({

9    name: z.string().describe('The skill name to load'),

10  }),

11  execute: async ({ name }, { experimental_context }) => {

12    const { sandbox, skills } = experimental_context as {

13      sandbox: Sandbox;

14      skills: SkillMetadata[];

15    };

16

17    const skill = skills.find(s => s.name.toLowerCase() === name.toLowerCase());

18    if (!skill) {

19      return { error: `Skill '${name}' not found` };

20    }

21

22    const skillFile = `${skill.path}/SKILL.md`;

23    const content = await sandbox.readFile(skillFile, 'utf-8');

24    const body = stripFrontmatter(content);

25

26    return {

27      skillDirectory: skill.path,

28      content: body,

29    };

30  },

31});
```

The tool returns the skill directory path alongside the content so the agent can construct full paths to bundled resources.

## [Step 5: Create the Agent](https://ai-sdk.dev/cookbook/guides/agent-skills\#step-5-create-the-agent)

Wire up the sandbox and skills using `callOptionsSchema` and `prepareCall`:

```ts
1const callOptionsSchema = z.object({

2  sandbox: z.custom<Sandbox>(),

3  skills: z.array(

4    z.object({

5      name: z.string(),

6      description: z.string(),

7      path: z.string(),

8    }),

9  ),

10});

11

12const readFileTool = tool({

13  description: 'Read a file from the filesystem',

14  inputSchema: z.object({ path: z.string() }),

15  execute: async ({ path }, { experimental_context }) => {

16    const { sandbox } = experimental_context as { sandbox: Sandbox };

17    return sandbox.readFile(path, 'utf-8');

18  },

19});

20

21const bashTool = tool({

22  description: 'Execute a bash command',

23  inputSchema: z.object({ command: z.string() }),

24  execute: async ({ command }, { experimental_context }) => {

25    const { sandbox } = experimental_context as { sandbox: Sandbox };

26    return sandbox.exec(command);

27  },

28});

29

30const agent = new ToolLoopAgent({

31  model: yourModel,

32  tools: {

33    loadSkill: loadSkillTool,

34    readFile: readFileTool,

35    bash: bashTool,

36  },

37  callOptionsSchema,

38  prepareCall: ({ options, ...settings }) => ({

39    ...settings,

40    instructions: `${settings.instructions}\n\n${buildSkillsPrompt(options.skills)}`,

41    experimental_context: {

42      sandbox: options.sandbox,

43      skills: options.skills,

44    },

45  }),

46});
```

## [Step 6: Run the Agent](https://ai-sdk.dev/cookbook/guides/agent-skills\#step-6-run-the-agent)

```ts
1// Create sandbox (your filesystem/execution abstraction)

2const sandbox = createSandbox({ workingDirectory: process.cwd() });

3

4// Discover skills at startup

5const skills = await discoverSkills(sandbox, [\
\
6  '.agents/skills',\
\
7  '~/.config/agent/skills',\
\
8]);

9

10// Run the agent

11const result = await agent.run({

12  prompt: userMessage,

13  options: { sandbox, skills },

14});
```

When a user asks something that matches a skill description, the agent calls `loadSkill`. The full instructions load into context, and the agent follows them using `bash` and `readFile` to access bundled resources.

## [Accessing Bundled Resources](https://ai-sdk.dev/cookbook/guides/agent-skills\#accessing-bundled-resources)

Skills can reference files relative to their directory. The agent uses existing tools to access them:

```markdown
1Skill directory: /path/to/.agents/skills/my-skill

2

3# My Skill Instructions

4

5Read the configuration template:

6templates/config.json

7

8Run the setup script:

9bash scripts/setup.sh
```

The agent sees the skill directory path in the tool result and prepends it when accessing `templates/config.json` or `scripts/setup.sh`. No special resource loading mechanism is needed—the agent uses the same tools it uses for everything else.
